# Backend Mastery: 60-Day Prep & Production Roadmap

Welcome to the **Backend Mastery** repository. This is a 60-day intensive program designed to take you from beginner concepts to interview-ready and production-ready senior backend engineering.

## 🚀 The Mission
This repository tracks my daily learning, assignments, and deep-dive notes as I prepare for high-stakes backend roles. No fluff, only industry-standard practices.

---

## 🗓️ 60-Day Backend Roadmap

### Phase 1: The Foundation (Days 1-5)
*   **Day 1**: How the Internet Actually Works (DNS, TCP/IP, Packets) ✅
*   **Day 2**: HTTP / HTTPS Deep Dive (Methods, Headers, Status Codes, TLS) ✅
*   **Day 3**: Node.js Internals (V8, libuv, Threads)
*   **Day 4**: The Event Loop (Macrotasks, Microtasks, Real-world blocking)
*   **Day 5**: Streams & Buffers (Handling large data without crashing)

### Phase 2: Building APIs & Express (Days 6-12)
*   **Day 6**: REST API Design Principles (Richardson Maturity Model, Versioning)
*   **Day 7**: Express.js Architecture & Middleware Patterns
*   **Day 8**: Advanced Routing & Controller Patterns
*   **Day 9**: Input Validation & Sanitization (Zod / Joi)
*   **Day 10**: Global Error Handling & Async Wrappers
*   **Day 11**: Production Logging (Winston/Pino) & Request Tracing
*   **Day 12**: Building a robust API Boilerplate (Mini-project)

### Phase 3: Database & SQL Mastery (Days 13-25)
*   **Day 13**: Relational DB Concepts & Normalization (1NF to 3NF)
*   **Day 14**: SQL Deep Dive: Complex Joins & Subqueries
*   **Day 15**: Indexes Under the Hood (B-Trees, Hash, Bitmap)
*   **Day 16**: Query Optimization & `EXPLAIN ANALYZE`
*   **Day 17**: ACID Properties & Transactions in Practice
*   **Day 18**: Concurrency, Locks, & Isolation Levels
*   **Day 19**: Node.js + PostgreSQL Integration (pg, Query Builders, ORMs)
*   **Day 20**: Connection Pooling & DB Reliability
*   **Day 21**: Migrations & Database Version Control
*   **Day 22**: NoSQL vs SQL: When to use MongoDB
*   **Day 23**: MongoDB Basics & Aggregation Pipeline
*   **Day 24**: Redis Basics (Data structures, TTL)
*   **Day 25**: Caching Strategies (Write-through, Cache-aside)

### Phase 4: Security & Authentication (Days 26-32)
*   **Day 26**: Authentication vs Authorization (Concepts)
*   **Day 27**: Session-based Auth & Cookies (Security flags)
*   **Day 28**: JWT Anatomy & Security Flaws (XSS, CSRF)
*   **Day 29**: Refresh Tokens & Blacklisting
*   **Day 30**: OAuth 2.0 & OpenID Connect Flows
*   **Day 31**: Role-Based Access Control (RBAC) & Middleware
*   **Day 32**: OWASP Top 10 API Security Risks

### Phase 5: Architecture & Advanced Systems (Days 33-42)
*   **Day 33**: MVC vs Clean Architecture / Onion Architecture
*   **Day 34**: Scalable Folder Structures & Dependency Injection
*   **Day 35**: File Uploads (Multer, S3, Presigned URLs)
*   **Day 36**: Background Jobs & Message Queues (BullMQ, RabbitMQ)
*   **Day 37**: Cron Jobs & Schedulers
*   **Day 38**: Sending Emails Reliably (Templates, Queues, SES)
*   **Day 39**: Real-time Web: WebSockets & Socket.io
*   **Day 40**: Real-time Web: SSE (Server-Sent Events) & Polling
*   **Day 41**: Payment Integrations (Stripe Webhooks & Idempotency)
*   **Day 42**: GraphQL Basics & N+1 Problem

### Phase 6: Scaling & Performance (Days 43-50)
*   **Day 43**: Rate Limiting & Throttling Algorithms (Token Bucket, Sliding Window)
*   **Day 44**: API Gateways & Reverse Proxies (Nginx basics)
*   **Day 45**: Load Balancing (L4 vs L7, Round Robin)
*   **Day 46**: Horizontal vs Vertical Scaling
*   **Day 47**: Microservices Basics (When NOT to use them)
*   **Day 48**: Event-Driven Architecture Basics
*   **Day 49**: Profiling Node.js (Memory Leaks, CPU Profiling)
*   **Day 50**: Production Debugging & Incident Response

### Phase 7: DevOps & Deployment (Days 51-54)
*   **Day 51**: Docker Basics (Images, Containers, Volumes)
*   **Day 52**: Dockerizing a Node.js + Postgres App
*   **Day 53**: CI/CD Basics (GitHub Actions)
*   **Day 54**: Monitoring & Observability (Prometheus, Grafana, APM)

### Phase 8: System Design Interviews (Days 55-60)
*   **Day 55**: System Design Framework (Requirements, Estimation, High-Level, Deep Dive)
*   **Day 56**: Design a URL Shortener (Bitly)
*   **Day 57**: Design a Chat App (WhatsApp/Discord)
*   **Day 58**: Design a Notification System
*   **Day 59**: Design a Feed System (Twitter/Instagram)
*   **Day 60**: Design an E-commerce Backend / Booking System
